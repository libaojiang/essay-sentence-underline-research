# 03 测试过程与效果

## 测试集

测试主要来自三个目录：

```text
tmp/语文作文素材
tmp/英语作文素材
tmp/普通测试集
```

覆盖场景：

| 类型 | 重点问题 |
| --- | --- |
| 语文作文 | 多页、方格纸、跨行长句、左右半页干扰 |
| 英语作文 | 单词间距大、行距不稳定、灰底、倾斜 |
| 普通测试集 | 非作文图片、题目图片、电脑截图、普通拍照图 |

## 测试方法

每次不是只看接口成功，而是保存三类产物：

1. 原始接口响应 JSON。
2. 根据返回坐标画线后的图片。
3. 中间调试图：二值化、文字 mask、行框、白带、候选句。

人工检查项：

- 分句点评是否只输出问题句。
- 点评内容和画线句子是否对应。
- 是否有画到完全无关句子的情况。
- 是否只画半句。
- 是否压字。
- 是否跨行。
- 是否画到左右半页干扰区域。
- 多图作文是否被合并成一篇。
- 英文作文是否用中文评价。
- 非作文图片是否被拦截。

## 关键样例：c6

`c6` 是重点样例，因为它有多页、左右半页干扰、作文纸网格、手写倾斜等问题。

最终画线效果：

![c6 最终效果](./assets/final_chinese_c6_page3_underlines.jpg)

早期容易出现的问题：

- OCR 把左侧或右侧半页也识别进正文。
- 分句点评选中的句子和画线坐标不是同一句。
- 线段水平画，和倾斜手写行不一致。
- 下划线压在字中间。

最终处理：

- 用主体区域和物理行过滤左右干扰。
- 分句点评只从可定位候选句里选。
- 坐标使用斜线。
- 用白带和墨迹检测避让。

## 响应时间观察

测试中发现耗时主要来自三块：

| 环节 | 说明 |
| --- | --- |
| 视觉转写 | 用于生成整篇作文正文，总评和追问修改依赖它 |
| OCR / 物理行识别 | 用于拿坐标和可定位候选句 |
| LLM 分句点评 | 从候选句里选问题句并生成点评 |

优化方向：

- 总评和分句点评拆成两个并发任务。
- 多图 OCR 并发。
- 分句点评只传候选句，不传全部 OCR 原文。
- 候选句最多保留必要数量，减少 LLM 输入。

## 测试结论

当前版本的判断标准不是“所有句子都能画”，而是：

```text
只要画出来，就必须和点评内容对应。
```

因此如果候选句不稳定，系统宁愿少返回一条分句点评，也不返回错误坐标。
