# 作文批改分句划线研究过程（求助帖）

大家好，我最近在做一个作文批改功能，里面有一个比较难处理的小点：模型不仅要给作文总评，还要挑出几句有问题的句子做点评，并且把这些句子在原图里的位置返回给前端画下划线。

我自己能力有限，前前后后试了很多版方案，目前流程勉强能跑通，但响应速度和识别/批改效果都还没有达到我想要的程度。所以我把这段时间的研究过程、demo、效果图和公式都整理到了这个文件夹里，想发出来请教一下各位前辈、老师或者做过类似方向的朋友。

如果大家有时间帮我看一眼，哪怕只指出其中一个问题，我也非常感谢。

## 当前遇到的问题

### 1. 响应时间偏慢

我现在的流程里，一张或多张作文图片进入批改后，通常要经过图片预处理、OCR/视觉识别、作文总评、分句点评、句子定位、坐标生成等多个步骤。为了尽量保证能返回可画线的坐标，我中间又加了物理行切分、二值化、行过滤、候选句匹配等处理。

这样做虽然能解决一部分定位问题，但整体链路被拉长了，响应时间不够理想。尤其是多页作文、图片质量较差、需要多次模型调用时，耗时会明显上升。

我现在比较困惑的是：这种功能如果要上线，到底应该牺牲一点定位准确率来换速度，还是继续拆步骤保证稳定性？有没有更成熟的工程做法？

### 2. 识别效果还不稳定

真实作文图片比我一开始想象得复杂很多，经常会遇到：

- 手机斜拍、透视变形、灰底、阴影、模糊。
- 作文纸有横线、方格、竖线，容易干扰文字检测。
- 一张图里可能拍到旁边半页，导致 OCR 把无关文字识别进来。
- 手写字可能出现漏识别、错识别、断句错误。
- 有时模型能理解文章，但无法稳定给出和原图完全对应的句子位置。

这些问题会影响后面的分句点评和下划线定位。

我试过直接用 OCR 坐标，也试过用视觉模型输出句子，再和 OCR/CV 结果匹配，还试过先切物理行再做候选句。每个方案都能解决一部分问题，但都不是特别稳。

### 3. 批改效果没有完全达到预期

我的目标不是简单输出一段作文评价，而是希望它同时做到：

1. 总体评价能看懂整篇作文。
2. 分句点评只评价真实存在的句子。
3. 点评内容和前端画线位置要一致。
4. 下划线不能乱画，不能画到完全无关的句子。

现在的问题是：总评大多数情况下还能用，但分句点评和定位仍然有不稳定情况。有时模型选出的句子不够完整，有时 OCR 候选句和模型点评句对不上，有时为了保证定位准确又会导致可点评句子变少。

也就是说，我现在比较尴尬：想让它多点评几句，就容易定位不准；想让它定位准一点，能点评的句子又变少。

## 我希望得到的帮助

我主要想请教几个方向。如果有朋友对其中任何一个点有经验，麻烦帮我指一下路：

- 手写作文图片，如何更稳定地提取正文区域和文本行？
- 在不显著增加响应时间的情况下，怎么提高句子识别和句子定位准确率？
- 分句点评应该先由视觉模型选句，再匹配坐标，还是先生成可定位候选句，再让模型从候选句里评价？
- 对于斜拍、灰底、方格纸、旁边半页干扰这些真实拍照问题，有没有更稳的工程方案？
- 如果要做成线上产品，哪些步骤值得保留，哪些步骤应该简化？

我不确定现在的方向是不是走偏了，也不知道有没有现成的 OCR/文档结构分析方案更适合这种场景。如果有更好的整体思路，也希望大家直接指出来。

## 研究目标

我这个作文批改功能需要输出两类结果：

1. 总体评价：总分、分项得分、总体评语、亮点、学习建议。
2. 分句点评：最多 3 条问题句，给出问题说明和润色，并返回图片坐标让前端画下划线。

难点在第二点：模型能指出哪句话有问题，但后端必须知道这句话在原图上的位置，否则前端没法画线。

## 当前阶段结论

我目前的阶段性判断是：单靠 OCR 或单靠视觉模型都不够稳定。现在尝试过的相对可行方向是拆分式流程：

```text
视觉模型负责整篇作文理解和总评
CV/物理行切分负责找到可定位文本行
单行识别负责得到可定位文字
LLM 只从可定位候选句里做分句点评
坐标算法负责斜线、避让和裁剪
```

核心原则：

- 总评优先保证文章理解质量。
- 分句点评优先保证能定位、能画线。
- 宁愿少画，也不要画到完全不相关的句子。
- 坐标返回保持简单，前端只负责按图片显示比例缩放。

但我也知道这个方案还不是最终成熟方案，主要瓶颈仍然是响应速度和真实拍照场景下的识别稳定性。

如果各位能帮我看看哪里设计得不合理，或者有没有更简单、更稳的方案，我会非常感谢。
