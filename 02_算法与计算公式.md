# 02 算法与计算公式

这一部分记录的是分句划线里最关键的图像算法：二值化、物理行检测、斜率计算、坐标还原和下划线避让。

## 1. 二值化的思考过程

最开始我以为二值化只是一个预处理步骤，后来发现它会直接决定后面能不能切准行。

作文图片里主要有四类东西：

1. 手写文字：真正要检测的前景。
2. 作文纸格线/横线/竖线：不是文字，但很容易被当成前景。
3. 背景灰度和阴影：手机拍照、打印纸、屏幕截图都会产生灰底。
4. 噪点和压缩痕迹：会干扰投影统计。

所以二值化的目标不是“让图片变黑白”这么简单，而是：

```text
尽量保留手写文字墨迹；
尽量去掉纸张背景、阴影、格线、噪点；
让 y 方向投影能稳定反映每一行文字的位置。
```

### 1.1 固定阈值的问题

最早尝试过固定阈值：

```text
foreground(x, y) = 1, if gray(x, y) < T
foreground(x, y) = 0, otherwise
```

这个在白底清晰图片上能用，但在真实样例里很快失败：

- 灰底图片会把大片背景识别成文字。
- 有阴影时，同一张图左边和右边亮度不同。
- 淡色手写字可能被阈值过滤掉。
- 深色格线会和手写字一起被保留下来。

结论：固定阈值不适合手机拍照作文。

### 1.2 全局 Otsu 的问题

也考虑过全局 Otsu 阈值：

```text
T_otsu = argmax_T sigma_between_class^2(T)
```

它会根据整张图自动选阈值，比固定阈值好一点，但仍然是全局阈值。遇到局部阴影、灰底和曝光不均时，局部文字仍然容易丢失。

结论：全局阈值仍然不够稳。

### 1.3 最终选择：局部自适应阈值 + 规则线剔除

最后采用的方向是：

```text
图片增强
-> 灰度化
-> 自适应二值化
-> 形态学检测横线/竖线
-> 从前景中减掉规则线
-> 得到 handwriting foreground mask
```

它的好处是：

- 局部阈值能适应灰底和阴影。
- 规则线剔除能避免作文纸格线影响行检测。
- 最终得到的 mask 更接近真实手写墨迹。

## 2. 灰度化公式

输入 RGB 图片：

```text
I_rgb(x, y) = (R(x, y), G(x, y), B(x, y))
```

先转灰度：

```text
I_gray(x, y) = 0.299 * R(x, y) + 0.587 * G(x, y) + 0.114 * B(x, y)
```

如果图片偏灰、对比度低，会先做轻量增强：

```text
I_enhanced = enhance(I_rgb)
I_gray = gray(I_enhanced)
```

增强不能太重，因为目标不是做漂亮图片，而是让文字墨迹和背景更容易分开。过度增强会把纸张纹理、阴影、压缩噪点也放大。

## 3. 自适应二值化公式

对每个像素 `(x, y)`，取它附近一个窗口 `W(x, y)`，计算局部阈值：

```text
T_local(x, y) = weighted_mean(I_gray(u, v), (u, v) in W(x, y)) - C
```

如果使用高斯权重，可以写成：

```text
T_local(x, y) =
    sum_{(u,v) in W} G_sigma(x-u, y-v) * I_gray(u, v)
    /
    sum_{(u,v) in W} G_sigma(x-u, y-v)
    - C
```

其中：

- `W` 是局部窗口。
- `G_sigma` 是高斯权重。
- `C` 是偏移量，用来控制前景保留强度。

二值化时，手写文字通常比纸张背景更暗，所以：

```text
B(x, y) = 1, if I_gray(x, y) < T_local(x, y)
B(x, y) = 0, otherwise
```

这里 `B(x, y) = 1` 表示可能是文字或线条的黑色前景。

代码里 OpenCV 的二值图通常是 `0/255`，为了公式清楚，这里统一写成 `0/1`。

## 4. 去除作文纸规则线

二值化后，作文纸横线、竖线也会变成前景。它们会让投影结果变得很密，导致行检测错位。

所以用形态学开运算检测长直线：

```text
H = open(B, K_h)
V = open(B, K_v)
```

其中：

```text
K_h = ones(1, w_h)
K_v = ones(h_v, 1)
```

- `H` 表示横向规则线。
- `V` 表示纵向规则线。
- `K_h` 是横向长核。
- `K_v` 是纵向长核。

合并规则线：

```text
Ruling = dilate(H OR V, K_d)
```

最终文字前景：

```text
F = B AND NOT Ruling
```

这里 `F` 才是后续行检测使用的文字墨迹 mask。

为什么要 `dilate` 一下规则线：

```text
格线本身可能只有 1-2 像素宽；
如果不膨胀，格线边缘残留会继续影响投影。
```

## 5. y 方向投影：找物理行

得到文字前景 `F` 后，对 y 方向做投影：

```text
R(y) = sum_{x=0}^{W-1} F(x, y)
```

为了消除单个噪点，做一维平滑：

```text
R_s(y) = (1 / (2r + 1)) * sum_{i=-r}^{r} R(y + i)
```

然后取高于阈值的连续区域：

```text
LineMask(y) = 1, if R_s(y) > theta_y
LineMask(y) = 0, otherwise
```

连续的 `LineMask=1` 区间就是候选物理行：

```text
line_j = [y1_j, y2_j]
```

相邻区间如果距离很近，需要合并：

```text
if y1_next - y2_prev <= merge_gap:
    merge(line_prev, line_next)
```

过滤太小的行：

```text
height(line_j) = y2_j - y1_j + 1
ink(line_j) = sum_{y=y1_j}^{y2_j} R(y)

keep(line_j) =
    height(line_j) >= h_min
    AND ink(line_j) >= ink_min
```

这一层的目标是找“真实写字行”，不是找完整句子。

## 6. x 方向投影：找每行左右边界

对每个物理行区域 `[y1, y2]`，继续计算 x 方向投影：

```text
C(x) = sum_{y=y1}^{y2} F(x, y)
```

有墨迹的 x 区间：

```text
Run = connected_segments(C(x) > 0)
```

如果有多个区间，需要选择主体。评分公式：

```text
score(run) = ink(run) + width(run) * alpha - center_penalty(run) * beta
```

其中：

```text
ink(run) = sum_{x in run} C(x)
width(run) = x2 - x1 + 1
center_penalty(run) = abs(center(run) - image_center) / image_center
```

这样做是为了处理左右半页干扰。旁边半页可能也有文字，但它通常不在主体中心，且宽度/墨迹分布不同。

最终得到行的左右边界：

```text
x1_j, x2_j = argmax_run score(run)
```

## 7. 斜率计算的思考过程

早期画线都是水平线：

```text
y1 = y2 = constant
```

但真实图片经常有轻微倾斜。水平线在倾斜手写行上会出现两个问题：

- 左边位置合适，右边可能压字。
- 或者右边位置合适，左边离文字太远。

所以需要给每一行单独计算斜率，而不是整张图统一旋转。

为什么不只算整页倾斜角：

```text
手机拍照会有透视变形；
纸张可能弯曲；
学生每一行书写本身也可能不完全平行；
整页一个角度不一定适合所有行。
```

因此最终采用“每行局部斜率”。

## 8. 墨迹点加权拟合斜率

对某一行的文字前景点集合：

```text
P = {(x_i, y_i)}
```

每个点有权重 `w_i`，通常可以简单取：

```text
w_i = 1
```

也可以根据局部墨迹强度赋权。

先计算加权均值：

```text
x_bar = sum(w_i * x_i) / sum(w_i)
y_bar = sum(w_i * y_i) / sum(w_i)
```

拟合直线：

```text
y = kx + b
```

加权最小二乘的斜率：

```text
k =
sum(w_i * (x_i - x_bar) * (y_i - y_bar))
/
sum(w_i * (x_i - x_bar)^2)
```

截距：

```text
b = y_bar - k * x_bar
```

为了防止异常点导致斜率过大，会限制范围：

```text
if abs(k) > k_max:
    k = 0 or fallback
```

实际使用时，小角度倾斜就足够了，不需要为了极端图片强行拟合很大的角度。

## 9. 四边形边缘斜率兜底

如果一行已经有 OCR 四边形，也可以从四边形估计斜率。

四边形点按顺序：

```text
top_left, top_right, bottom_right, bottom_left
```

上边斜率：

```text
k_top = (top_right.y - top_left.y) / (top_right.x - top_left.x)
```

下边斜率：

```text
k_bottom = (bottom_right.y - bottom_left.y) / (bottom_right.x - bottom_left.x)
```

最终取中位数或平均值：

```text
k = median(k_top, k_bottom)
```

这个适合作为 OCR 框已有斜率时的补充。

## 10. 行四边形计算公式

已知：

```text
x1, x2       行左右边界
y_center     行中心 y
x_center     行中心 x
h            行高度
k            行斜率
```

行中心线：

```text
y_mid(x) = y_center + k * (x - x_center)
```

上边界：

```text
y_top(x) = y_mid(x) - h / 2
```

下边界：

```text
y_bottom(x) = y_mid(x) + h / 2
```

四边形：

```text
top_left     = (x1, y_top(x1))
top_right    = (x2, y_top(x2))
bottom_right = (x2, y_bottom(x2))
bottom_left  = (x1, y_bottom(x1))
```

这就是单行文字在图片上的斜框。

## 11. 下划线端点公式

下划线应该在文字下方一点，而不是穿过文字中心。

给定偏移量 `d`：

```text
y_under(x) = y_bottom(x) + d
```

如果要画整行：

```text
P1 = (x1, y_under(x1))
P2 = (x2, y_under(x2))
```

如果只画句子的一部分，先得到句子在该行里的左右位置：

```text
xs, xe = text_span_x_range(sentence_segment)
```

则：

```text
P1 = (xs, y_under(xs))
P2 = (xe, y_under(xe))
```

展开后：

```text
P1 = (xs, y_center + k * (xs - x_center) + h / 2 + d)
P2 = (xe, y_center + k * (xe - x_center) + h / 2 + d)
```

所以前端收到的 `x1/y1/x2/y2` 可以天然表示斜线。

## 12. 裁剪坐标还原

如果先裁剪主体区域，再在裁剪图上检测行，必须还原到原图坐标。

裁剪框：

```text
crop = [crop_x1, crop_y1, crop_x2, crop_y2]
```

裁剪图坐标还原：

```text
x_original = x_crop + crop_x1
y_original = y_crop + crop_y1
```

如果有缩放：

```text
x_original = x_crop * scale_x + crop_x1
y_original = y_crop * scale_y + crop_y1
```

其中：

```text
scale_x = original_width / processed_width
scale_y = original_height / processed_height
```

这是保证“后端用处理图识别，前端在原图画线”不出错的关键。

## 13. 白带避让公式

即使斜率正确，下划线仍可能压到字。解决办法是在附近找更干净的横向区域。

对候选下划线 `L(y)`，沿线附近采样一个窄带：

```text
Band(L, r) = pixels whose distance to L <= r
```

墨迹密度：

```text
D(L) = sum(F(x, y), (x, y) in Band(L, r)) / area(Band(L, r))
```

在候选偏移集合里选最干净的线：

```text
L_best = argmin_{delta in Delta} D(L_delta)
```

其中：

```text
L_delta: y = y_under(x) + delta
Delta = {-a, ..., 0, ..., b}
```

约束：

```text
delta 不能让线跨到下一行正文；
下移最多执行一次；
如果移动后仍压字，则宁愿丢弃该坐标。
```

## 14. 局部 x 范围收缩

有时一行的框很长，但目标句子只占其中一部分。为了避免线画过长，会按局部墨迹收缩：

```text
x_start = first_x_with_ink - margin
x_end   = last_x_with_ink + margin
```

但收缩必须有限制：

```text
new_width = x_end - x_start
old_width = xe - xs

if new_width < max(old_width * 0.12, min_width):
    keep_old_range
```

这样避免噪点导致线段变得太短。

## 15. 可画性判断

最终坐标必须通过一组检查：

```text
line_width >= min_width
0 <= x1, x2 <= image_width
0 <= y1, y2 <= image_height
not hits_handwriting(line)
inside_main_body(line)
```

其中 `hits_handwriting(line)` 本质上还是墨迹密度判断：

```text
hits = D(line) > density_threshold
```

如果检查失败，就不返回这条坐标。

最终策略是保守的：

```text
少画一条可以接受；
画到完全不相关的句子不可接受。
```
